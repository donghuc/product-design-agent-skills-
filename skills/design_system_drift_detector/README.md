# Design System Drift Detector
**ID:** `design_system_drift_detector`
**Category:** Design Ops & Governance

## Overview
A strict, deterministic auditor that diffs built Figma files or code against your official Design System definitions. It catches unauthorized overrides and custom tokens.

## When to Trigger
- **Phase 4 (DS Application)**
- Run after a designer skins a prototype to ensure they haven't "drifted" from the approved UI Guideline.

# UX Strategy Guardian

**ID:** `ux_strategy_guardian`
**Category:** Design Governance

## Overview

The UX Strategy Guardian is a specialized agent skill that acts as a gatekeeper in a multi-agent product design workflow. It evaluates UX flows and UI proposals generated by other agents to ensure they align with the overarching product vision, UX strategy, established design principles, and design system rules.

**It does not generate UI.** It only evaluates, critiques, and provides constraints for the next iteration.

## When to Trigger

Trigger this skill immediately after an agent proposes a new UX flow, UI component, or interface layout, but before that proposal is finalized or handed off to development.

## Special Features

- **Foundation Mode**: If the triggering agent fails to provide a `product_vision` or `ux_strategy` in the input, the Guardian will switch to Foundation Mode. It will draft a basic, context-aware UX foundation and request confirmation before proceeding with strict evaluation.

## Inputs Schema

The skill expects a JSON payload containing context about the product, rules, and the actual proposed design.

- `product_foundation` (Object): The strategic vision and principles.
- `design_system_rules` (Object): Constraints from your design system (components, tokens).
- `project_context` (Object): The specific feature or platform being designed.
- `proposal` (Object | String): The UI/UX solution to evaluate (could be text describing a flow, or a link to a Figma node).

*See the [examples/](./examples) directory for concrete payload examples.*

## Expected Output

The agent will return a structured JSON response evaluating the alignment and listing any rule violations, along with recommendations for the next iteration.
